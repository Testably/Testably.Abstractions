# Intercept & Notify

`MockFileSystem` exposes two hooks for reacting to file system activity in tests:

- **`Intercept`** runs **before** an operation completes and can throw to abort it.
- **`Notify`** runs **after** an operation completes and supports awaiting the next event.

## Intercept — abort an operation

```csharp
MockFileSystem fileSystem = new();

fileSystem.Intercept.Creating(FileSystemTypes.File,
    _ => throw new Exception("my custom exception"));

// All file creations now throw "my custom exception"
fileSystem.File.WriteAllText("a.txt", "x");
```

Useful for simulating disk errors, permission failures or race conditions.

## Notify — await an event

`Notify` returns an awaitable handle. It removes the subscription on dispose and lets you block the test until the event fires:

```csharp
MockFileSystem fileSystem = new();

fileSystem.Notify
    .OnCreated(FileSystemTypes.File, _ =>
    {
        // do something on every file creation
    })
    .ExecuteWhileWaiting(() =>
    {
        // anything inside this lambda triggers the callback above
        fileSystem.File.Create("some-file.txt");
    })
    .Wait();
```

Use this to deterministically test code that fires off async work and then writes to the file system — the test can wait for the write to actually happen instead of polling or sleeping.
