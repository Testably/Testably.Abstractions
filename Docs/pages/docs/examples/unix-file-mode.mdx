# Unix file mode

> Source: [Examples/UnixFileMode](https://github.com/Testably/Testably.Abstractions/tree/main/Examples/UnixFileMode)

This example highlights how to simulate working with [`UnixFileMode`](https://learn.microsoft.com/en-us/dotnet/api/system.io.unixfilemode) in the `MockFileSystem`.

The `DefaultUnixFileModeStrategy` simulates the user and group, and stores this information alongside the file/directory in the `OnSetUnixFileMode` method. Later, in `IsAccessGranted`, you can access the stored data from `IFileSystemExtensibility` and decide whether to grant read or write requests.

Read access is granted when:

- the unix file mode is set to `OtherRead`, **or**
- the unix file mode is set to `GroupRead` and the simulated group of the file matches the currently simulated group, **or**
- the unix file mode is set to `UserRead` and the simulated user of the file matches the currently simulated user

The same logic applies to write access.

Simulate a user or group by calling `SimulateUser` / `SimulateGroup` on `DefaultUnixFileModeStrategy`.
