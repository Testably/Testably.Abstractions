# Thread-aware time provider

> Source: [Examples/ThreadAwareTimeProvider](https://github.com/Testably/Testably.Abstractions/tree/main/Examples/ThreadAwareTimeProvider)

The default implementation of `ITimeProvider` in `Testably.Abstractions.Testing` uses a `DateTime` property to simulate the current time, which is advanced with every `Thread.Sleep` or `Task.Delay` call.

In a multi-threaded scenario, parallel threads influence each other differently in the mocked instance than in the real world: two threads each sleeping for 5s should advance the total clock by 5s, but the default mocked instance advances it by 10s.

This example shows how to implement a thread-aware time provider for that scenario.

`ThreadAwareTimeProvider` stores the time in an `AsyncLocal<DateTime>` and provides a `SynchronizeClock` method to align the clock across multiple threads. The corresponding unit tests demonstrate the correct behaviour.

To use it, inject the custom provider into `MockTimeSystem`:

```csharp
MockTimeSystem timeSystem = new(new ThreadAwareTimeProvider());
```
